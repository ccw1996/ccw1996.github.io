  • 语言模型的训练时间：从估算到 FLOPs 推导

先说结论：

训练时间=6TP/(nX)

其中，T 是总 token 数，P 是参数量，n 是显卡数量，X 是 FLOPS（每张卡每秒实际做的浮点运算数，一般在理论上限的50%以上，可以通过samples/s换算，后文有提供计算方式。）

有一个前提：您没有开启激活值重计算。激活重计算目前有两种方式：full和selective。如果是full这个数字应该是8不是6。如果是selective差别不大。full其实浪费了一部分算力用时间换空间，所以不太推荐。

FLOPs 估算

对于每个step，模型的前向传播 FLOPs = 2∗batchsize∗seqlen∗参数量2 * batch_size * seq_len * 参数量2∗batchs​ize∗seql​en∗参数量

反向传播的计算量是前向传播的2倍(对于 h = XW，需要分别求出X的导数和W的导数)

所以，一次完整的前向 + 后向的 FLOPs = 6∗batchsize∗seqlen∗参数量6 * batch_size * seq_len * 参数量6∗batchs​ize∗seql​en∗参数量

那么总共需要训练 steps 步，完整的 FLOPs = 6∗steps∗batchsize∗seqlen∗参数量6 * steps * batch_size * seq_len * 参数量6∗steps∗batchs​ize∗seql​en∗参数量

其中 $steps * batch_size * seq_len￥ 就是语料的 Token 数量 T。

最后我们就得到了 FLOPs=6∗T∗PFLOPs = 6 * T * PFLOPs=6∗T∗P，也就是本文刚开头的分子部分

V词表数
  • Attention
    • key, query, value transformation
      • 输入的 QKV 三个矩阵 (B, s, h) 需要分别经过 Wq, Wk, Wv (h, h) 的线性变换：总共 FLOPs 为 6Bsh∗h6Bsh*h6Bsh∗h
    • attention matrix computation
      • Q 矩阵和 K 矩阵相乘，得到维度为 (s, s) 的 attention 矩阵：总共 FLOPs 为 2Bs∗sh2Bs*sh2Bs∗sh
    • attention over values
      • V 矩阵 (B, s, h) 和经过 softmax 的 attention 矩阵 (s, s) 相乘。总共 FLOPs 为 2Bs∗sh2Bs*sh2Bs∗sh
    • post-attention linear projection
      • 输出的矩阵还需要经过一次 Wo (h, h) 线性变换 ，总共 2Bsh∗h2Bsh*h2Bsh∗h
  • FFN
    • h -> 4h, 4h->h：总共为 16Bsh∗h16Bsh*h16Bsh∗h
    • LLaMa 等模型使用 swiglu 激活函数，同时多了一次线性变换，但是通过合理设置隐藏层大小将计算量保持不变。

把这些全加起来，每层FLOPs： 24Bsh∗h+4Bs∗sh24Bsh*h+4Bs*sh24Bsh∗h+4Bs∗sh

  • language model head
    • 输出矩阵(B, s, h) 需要乘上 lm head (h, V)： 2BshV

所以，整个语言模型的每个step的前向传播 FLOPs： l(24Bsh∗h+4Bs∗sh)+2BshVl(24Bsh*h+4Bs*sh)+2BshVl(24Bsh∗h+4Bs∗sh)+2BshV

按照刚才说的，反向是前向的2倍。所以每个step总共FLOPs： 72Bslh∗h+12Bs∗slh+6BshV72Bslh*h+12Bs*slh+6BshV72Bslh∗h+12Bs∗slh+6BshV

模型参数量计算

  • Layers
    • Attention
      • Wq, Wk ,Wv ,Wo: 4 * (h, h)
    • FFN
      • W(h -> 4h): (h, 4h)
      • W(4h -> h): (4h, h)
    • Layernorm * 2: (h,)
  • Embedding: (v, h)
  • LM Head: (h, v)
  • Final Layernorm: (h,)

加起来： 12lh∗h+(2l+1)h+2hV12lh*h+(2l+1)h+2hV12lh∗h+(2l+1)h+2hV
